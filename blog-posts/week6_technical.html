Blocs, Procs, and Lambdas

Blocs, Procs, and Lambdas are all a way of grouping code together. With procs and lambdas you can call on that block of code later because it is an object. Blocks are not objects so they cannot be called, they are just part of the method syntax. Below are a few examples of a block. We use them all the time already.

[1,2,3].each {|x| x * x} | {|x| x * x} is the block

[1,2,3].each do |x|
x * x
end
Between do to end is the block

Procs and lambdas have a similar purpose but differ in a few different ways. Shown below is a basic example of how to construct a proc and a lambda.

p = Proc.new {|x| x * x}
[1,2,3].each(&p)

Notice how we have to have the & symbol before we call the proc. This tells ruby it is time to turn the proc into a block.

p = Proc.new { puts "Hello World" }
p.call

See because procs are objects we can call methods on them like we do above. Here we call the proc and get the expected output.

lam = lambda {|x| x * x}
[1,2,3].each(&lam)

lam = lambda { puts "Hello World" }
lam.call

Almost the same format for both a lambda and a proc. Notice how there is no .new when creating the lambda, however. So how are they different? For one, lambdas will return an argument error if the wrong number of arguments are passed into them

lam = lambda { |x| puts x }
lam.call(2) #=> 2
lam.call #=> ArgumentError: wrong number of arguments (0 for 1)
lam.call(1,2,3) #=> # ArgumentError: wrong number of arguments (3 for 1)

p = Proc.new { |x| puts x }
p.call(2) #=> 2
p.call #=> nil
p.call(1,2,3) #=> 1

Notice when the proc recieves more arguments than it should, it would execute the proc on the first argument and forget about the rest. Lastly, when you use return in lambda it exits the block handing control to the calling method. With the proc it behaves like it is part of the calling method. Return with a proc would then return for both the block and the calling method.

def proc_test
proc = Proc.new { return }
proc.call
puts "Hello world"
end
proc_test #=> nothing is printed

def lambda_test
lam = lambda { return }
lam.call
puts "Hello world"
end
lambda_test #=> 'Hello World'
