SQL vs. NoSQL Databases

For this weeks technical blog I chose to write about the difference between SQL and NoSQL DBMSs. SQL servers have been around since the 1970s when programs started requiring ways to store and access data. With SQL servers, their records are stored in tables, with rows and columns storing a specific piece about that record. This system structure does have its drawbacks though. When trying to process a wide array of requests or when you want to make your servers work faster, you have to upgrade the server hardware instead of just adding additional servers. This new hardware is frequently expensive and can lead to lengthy server downtime.

[1,2,3].each {|x| x * x} | {|x| x * x} is the block

[1,2,3].each do |x|
x * x
end
Between do to end is the block

Procs and lambdas have a similar purpose by differ in a few different ways. Shown below is a basic example of how to construct a proc and a lambda.

(1..5).map {|x| (x * x) + 2} #=> [3, 6, 11, 18, 27]

.map is also useful on an array of strings. Lets say you wanted to capitalize every element in an array

["sam","fred","david"].map {|x| x.capitalize} #=> ["Sam", "Fred", "David"]

A super clever thing that people like to do is pass in a callable proc object, with a method, to run on each element in the array. By doing this you can write your function on a separate line in the proc, then call it on your array using .map

triple = Proc.new{|n| n*3}
[1,2,3].map(&triple) #=> [3, 6, 9]

Lastly, if you want to call a method instead of a proc, you can do this by using the METHOD method (confusing but look below)

def triple(n)
n * 2
end
[1,2,3].map(&method(:triple)) #=> [3, 6, 9]

You can use .map destructively by placing a '!' at the end of it. This will modify the original array though so be careful before using. Here I also show you how you could convert numbers into strings using .map

number_string = [1,2,3,4]
number_string.map!(&:to_s) #=> ["1", "2", "3", "4"]
